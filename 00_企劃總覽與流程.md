## 臨床資料重建行動 — 企劃總覽與流程

> 本版內容與任務網站（[yangrex0626.github.io/dentalcamp](https://yangrex0626.github.io/dentalcamp/)）完全同步，作為活動籌備、現場帶隊與工作人員操作的總覽文件。

## 一、活動概念

「臨床資料重建行動」是一套**十關逃脫室風格的解謎課程**，主軸是「思維挑戰」而非速度競賽。故事設定：一名代號 **A19** 的病患病歷遺失，各小組扮演「臨床資料重建小組」，依序破解十份原始資料／檢驗數據，重建出完整病歷，最終產出一份綜合診斷報告，找出 A19 口腔狀況半年內急速惡化的真正原因。

十份證物橫跨口腔醫學十個子領域：胚胎組織學、牙體型態學、牙體組織學（加密）、口腔解剖學、生物化學、牙周病學、化學（pH 計算）、牙科材料學、臨床急症處理（互動決策樹）、口腔顎面放射線學。每一關都要求學員實際運用當關提供的背景知識做出推理，答案無法用猜測或試誤取巧算出。

**全程採「正數計時」（碼表模式），不設時間上限**——重點是推理過程經得起檢驗，不是比速度快慢。

## 二、適合對象與分組建議

- 對象：高中／大學口腔醫學營隊學員，具備基礎生物化學／解剖學背景尤佳，但每一關都附有完整背景知識，無先備知識也能作答。
- 建議每組 3–5 人，人數太少討論深度不足，太多則容易有人閒置。
- 每組共用一個瀏覽器分頁即可（手機、平板、筆電皆可，網站已做響應式優化）。

## 三、活動流程時間軸（建議）

| 階段 | 內容 | 建議時間 |
|---|---|---|
| 開場簡報 | 說明故事背景、解謎慣例、如何使用 SOS 提示 | 5–10 分鐘 |
| 隊名輸入 | 各組於任務網站輸入隊名，正式開始計時 | 2 分鐘 |
| 十關解謎 | 依序破解證物①–⑩，過關後自動釋出下一段劇情 | 60–120 分鐘（依組別調整） |
| 結案 | 全數過關後顯示完整診斷報告，下載任務完成證書 PDF | 5 分鐘 |
| 總結／講評 | 工作人員統一講解正解與各關鍵知識點 | 10–15 分鐘 |

十關整體難度為漸進式，證物⑨（外傷急救，互動決策樹）與證物⑩（影像判讀）安排在最後，作為統整前面知識點的高難度收尾。

## 五、任務網站功能總覽

- **靜態網站**，部署於 GitHub Pages，免安裝、免登入，開瀏覽器輸入網址即可使用：`https://yangrex0626.github.io/dentalcamp/`
- **手機／平板／桌機皆已做響應式優化**，現場用手機答題也能正常顯示。
- **正數計時器**：從按下「開始任務」起算，畫面右上角即時顯示經過時間（超過一小時會自動切換為 H:MM:SS 格式）。
- **SOS 提示（次數不限）**：每關提示會依序累加顯示，越晚要越多提示，但不限制使用次數；提示內容經過設計，會引導思路而非直接洩題。
- **進度列**：頂部圓點即時顯示各關卡完成狀態（已完成／進行中／未開始）。
- **證物③加密解謎**：需要學員自行完成凱薩加密（循環金鑰）的手動解密，題目已附解密範例。
- **證物⑤即時圖表**：Stephan 曲線由網站即時繪製 SVG 圖表，不需另外準備圖檔。
- **證物⑨互動決策樹**：以六個連續判斷節點呈現外傷處置流程，選對才會往下走並取得一位數字，選錯只顯示後果、不洩漏正解，全部選對後系統自動組成六位數驗證碼。
- **證物⑩根尖片示意圖**：四張 X 光示意圖由網站即時繪製，標示 CEJ–齒槽骨嵴距離。
- **結案報告**：十關全數過關後，自動顯示完整診斷報告（FINAL_REPORT_HTML），統整六大致病因子。
- **任務完成證書**：結案畫面提供「下載證書 PDF」按鈕，前端直接產生證書 PDF 供下載（不透過瀏覽器原生列印功能，避免不同裝置排版跑版的問題），證書含營隊 Logo、隊名、完成時間、證物完成度、簽名等資訊。
- **本機進度保存**：答題進度存在瀏覽器 `localStorage`，中途重新整理頁面不會遺失進度。
- **測試結果自動回報**：學員完成任務時，系統會透過 EmailJS 自動寄送隊名、完成度、SOS 使用次數、裝置資訊等到工作人員信箱，全程不需要學員手動操作，畫面上也不會顯示任何提示（詳見《02》操作說明章節）。

## 六、工作人員專用操作說明

以下功能僅供工作人員使用，多數入口是畫面下方不顯眼的文字連結，避免學員誤觸；需要密碼的操作皆使用同一組密碼 **940626**：

| 功能 | 入口位置 | 說明 |
|---|---|---|
| 提前結束本隊任務 | 遊戲畫面下方「工作人員專用：提前結束本隊任務」 | 需輸入密碼確認，直接跳至結案畫面 |
| 重置任務（遊戲中隨時可用） | 遊戲畫面頂部（SOS 按鈕與計時器之間）「重置任務」 | 需輸入密碼確認，會立即清除本機進度、回到隊名輸入畫面，方便工作人員測試或排除問題時快速重來，不需要先破關或按其他按鈕 |
| 重置任務（換下一隊使用） | 結案畫面下方「重置任務（下一隊使用）」 | 需連續按兩次才會生效，清除本機進度，讓同一台裝置可以給下一組重新開始 |
| 跳至指定關卡 | 開場畫面或遊戲畫面下方「工作人員專用：跳至指定關卡」 | 需輸入密碼，再輸入要跳到第幾關（1–10），可直接跳關檢查內容，前面關卡會自動標記完成，不需要重新答題 |

> 提醒：以上「密碼」與「連按兩次確認」都只是避免學員誤觸的簡易防呆機制，原始碼是公開的，不能用來保護真正機密資訊。

## 七、技術規格與部署資訊

- 純前端靜態網站（HTML／CSS／原生 JavaScript），無後端伺服器、無資料庫。
- 部署平台：GitHub Pages，儲存庫 `yangrex0626/dentalcamp`，每次推送到 `main` 分支後自動重新建置上線。
- 自動回報信件：使用 [EmailJS](https://www.emailjs.com/) 免費方案，於前端直接呼叫寄信 API，設定值集中於 `data.js` 檔案最上方（`EMAILJS_PUBLIC_KEY`／`EMAILJS_SERVICE_ID`／`EMAILJS_TEMPLATE_ID`），已自行打包 SDK（`assets/emailjs.min.js`），不依賴外部 CDN。
- 證書 PDF：使用 html2canvas + jsPDF（皆已自行打包於 `assets/` 資料夾），前端直接截圖轉出 PDF，不經過瀏覽器列印功能。
- 簽名手寫字體：自行打包 Great Vibes 字型（`assets/GreatVibes-Regular.ttf`，SIL Open Font License），不依賴外部字型 CDN，離線或營隊現場網路不穩時仍可正常顯示。

## 八、注意事項與常見問題

- 若場地仍有舊版證物卡（少於十關的版本），請一併回收或更新，避免各組難度不一致。
- 現場建議至少準備 1–2 台備用裝置，避免單一裝置故障影響進度（進度存在瀏覽器本機，換裝置需重新開始）。
- 若需要協助某組除錯或提前檢查某一關內容，請使用「跳至指定關卡」功能，不需要請該組從頭重新作答。
- 自動回報信件僅寄送給工作人員信箱，收集內容為隊名、完成度、SOS 次數、裝置類型等技術資訊，不會收集學員個資或 IP 位址。
